# 设计稿 · Category Portfolio Diagnostic（类目货盘诊断）

> 状态：**已实现**（2026-06-15）。落地为 `references/17-category-portfolio-diagnostic.md` + `generate_report.py` 的 `report_type=category_portfolio` 渲染路径 + 示例 `examples/aurora-airpurifier-category-config.json`（虚构品牌/类目）。本稿保留为设计记录。
> 本稿用中文，便于审阅。

---

## 1. 它解决什么 / 何时触发

回答一个**类目操盘官**的问题：「我这个类目在这个国家，现在打得怎么样、哪里在漏血、每个型号下一步该怎么走？」

**路由分叉（写进 SKILL.md 决策树最顶部）：**

```
用户带着什么来？
├─ 单个 SKU + 国家（"华为 AX3 在罗马尼亚该怎么投"）
│     → 直接进现有 product × country pipeline（ref 13）+ HTE 深析
│
└─ 一个类目 + 国家（+ 一串在售型号清单）
      （"华为路由器在罗马尼亚整盘怎么操"）
      → 先走本 ref 17 类目诊断
      → 诊断选出「值得加大投入」的 SKU
      → 这些 SKU 再下沉到 ref 13 + HTE 深析
```

**ref 17 是 ref 13 的上游，不是替代。** 类目诊断画战略底图；ref 13 对选出的主力型号出执行计划。两层各做各的，互不混。

---

## 2. 因果闭环（核心原则）

```
诊断 Finding ──→ 代价 Implication ──→ 4P 建议 ──→（加大投入的 SKU）实验验证 ──┐
   ↑                                                                            │
   └────────────────────── 验证结果回灌，刷新诊断 ←──────────────────────────┘
```

铁律两条：
- **每条 4P 建议必须指回它要修补的那个诊断**——没有指向的建议是拍脑袋。
- **每个诊断必须指回它的证据**——没有证据的批评是耍嘴皮。

**先审后策。** 报告永远是「评价在前，建议在后」。

---

## 3. 输入契约（谁提供什么、什么置信度）

| 来源 | 内容 | provenance 默认 |
|------|------|----------------|
| **用户必填** | 品牌、类目、国家、在售型号清单 | — |
| **用户选填（强烈建议）** | 各型号出货量/销售额、毛利率、渠道合同、库存水位 | `Sourced` |
| **AI 公开扫描** | 公示价/促销价趋势、评论数+评分趋势、竞品型号+定价、评论里的 Push/Pull/Anxiety 信号、官网/PDP 是否存在 | `Hypothesis`（评论量 ≠ 销量；可核验的如"PDP 缺失"可升 `Sourced`） |

**Step 流程：**
```
Step 1  AI 扫描 → 出诊断草稿（6 镜 + 严重度，严重度受证据封顶）
Step 2  用户改错 / 补内部数据（薄证据的「观察」项重点确认）
Step 3  每 SKU：短诊断（几句话）→ 4P 一句话 → 裁决
Step 4  用户勾选「加大投入」的 SKU
Step 5  这些 SKU 下沉到 ref 13 + HTE 深析
```

## 4. 诊断框架：6 个审视镜（前 2 看市场，后 4 审 4P）

诊断**以类目总体为主**（这是报告主体），每个 SKU 只在矩阵里给几句话。

| 镜 | 审什么 | 尖锐问法举例（示意，非真实数据） |
|----|--------|------------------------------|
| **L1 价格带覆盖 & 白空间** | 钱在哪个带 vs 你在哪个带 | 「150–200€ 是增长最快的带，你一个 SKU 都没有，整带让给了竞品」 |
| **L2 品类动能** | 饼在涨还是缩、哪个带在涨 | 「入门带三年缩水，你的主力却全压在这里」 |
| **L3 货盘架构（Product）** | 死旗舰 / 僵尸 SKU / 断档 / 自相残杀 | 「X3 PRO 与 AX3 同价同定位，渠道二选一只会两败」 |
| **L4 价格站位（Price）** | 防御性 vs 无人区 vs 促销死亡螺旋 | 「NVWA 常年挂 -20%，锚价已打废，全价再也卖不动」 |
| **L5 渠道布点（Place）** | 是否在客群真正下单的货架 / 集中度风险 | 「移动热点在 eMAG 主推，但这品类 80% 销量在运营商，你在错的货架上」 |
| **L6 需求生成 & 数字货架（Promotion）** | 信息对不对人 / 官网·PDP·评论·本地化 | 「WiFi 7 旗舰没有罗语 PDP、零评论——上市即隐身」 |

**审策同构**：L3 审出来的病，就由 Product 那一句药来治；L4→Price，L5→Place，L6→Promotion。审完即得建议的因。

每个价格带还要带一层**人群 × 营销特性**（Four Forces 的段位版，AI 草拟、用户改）：

| 价格带 | 典型客群 | 决策驱动 | 适配打法（Hypothesis，待确认） |
|--------|---------|---------|------------------------------|
| 旗舰 | 技术尝鲜/品牌忠诚 | Pull 为主 | 品牌媒体、评测、建认知 |
| 中端 | 主流换机 | Push + Anxiety | 比价平台、KOL 横评、内容 |
| 入门 | 价格敏感首购 | Habit 阻力大 | 电商内部竞价、捆绑、首购优惠 |
| 移动热点 | 商旅/备用网 | 场景驱动 | 运营商渠道、B2B 采购 |

## 5. 让批评可信、而非嘴硬的铁律

> **严重度 ≤ 证据等级（Severity capped by evidence）**

| 严重度 | 含义 | 证据门槛 |
|--------|------|---------|
| 🔴 严重（Critical） | 此刻在丢钱/丢份额 | 必须 **Sourced**（可核验，如官网根本不存在） |
| 🟠 重大（Major） | 重大风险 | **Assumed** 或强 Hypothesis |
| 🟡 观察（Watch） | 已标记、证据薄 | **Hypothesis** → 自动进测试队列，不许冒充结论 |

一句很狠但只有薄证据的话（如"营销打错人群"但没有人群数据），**不能挂严重，自动降为观察 + 进队列**。刻薄必须缴证据税——这是我们和"嘴硬的咨询 PPT"的根本区别，也接上 ref 00b、ref 14、ref 15 Rule 10。

## 6. 一条诊断的标准结构（示意，非真实数据）

> 🟠 **重大 · WiFi 7 旗舰（WUKUN BE3）上市即隐身**
> **诊断**：eMAG 无罗语 PDP、评论数 0、官网无产品页。
> **证据**：PDP 缺失 `Hypothesis`（可核验→可升 Sourced）；评论数 0 `Sourced`。
> **代价**：高溢价新带里消费者搜不到、不敢买，认知份额拱手让给竞品首发。
> **→ Promotion**：旗舰不打促销，打技术媒体评测 + 建罗语 PDP。**→ Place**：极少量首发渠道即可，但货架必须有内容。

## 7. 报告输出结构

**第一屏 = SKU 裁决 + 4P 一句话矩阵**（一眼看完结论）：

| SKU | 裁决 | Product | Price | Place | Promotion |
|-----|------|---------|-------|-------|-----------|
| AX3 | **加大投入** | 中端主力，全屋覆盖 | 守 99–129€ | eMAG 主攻 | 场景内容打旧机痛点 |
| NVWA Mesh 3+ | **加大投入** | 大户型差异化 | 停常态促销，重建锚价 | 独立站+旗舰店 | 安装体验视频降 Anxiety |
| WS5203 AX1 | **维持现状** | 入门价格武器 | 对标竞品 -10% | 电商竞价+捆绑 | 秒杀，不讲故事 |
| WUKUN BE3 | **加大投入** | WiFi 7 建品牌高度 | 旗舰溢价不打折 | 极少量首发 | 评测稿+建 PDP |
| E5586 | **收割利润** | 移动热点，运营商业务 | 跟合约 | 运营商独占 | 流量捆绑，撤出 eMAG 主推 |
| X3 PRO | **计划退市** | 与 AX3 自相残杀 | — | — | 清库存，让位 AX3 |

**裁决词（完整中文，不玩哲学）：**
- **加大投入** — 有利润/增长空间，值得资源倾斜，进 Step 5 深析
- **维持现状** — 守住当前价格带，不追加投入
- **收割利润** — 停止投入、保毛利、安排接替款
- **计划退市** — 准备退出/清仓，已需接替款

**之后的章节顺序：**
1. 类目总体诊断（6 镜，主体，尖锐 + 证据 + 严重度）
2. 价格带 × 人群 × 竞品地图（每带一张）
3. 每 SKU 短诊断（几句话）+ 4P 建议 + 裁决 ← 第一屏矩阵的展开
4. 证据与缺口（沿用现有 §16 provenance 台账）
5. （按需）对「加大投入」SKU 的深析入口 → 交棒 ref 13

  "price_tiers": [
    {
      "id": "entry", "label": "入门 <99€", "trend": "shrinking",
      "audience": "价格敏感首购", "force": "Habit",
      "channel_fit": "电商竞价+捆绑",                 // Hypothesis

    }
  ],
  "portfolio": [
    {
      "sku": "AX3", "tier": "mid",
      "lifecycle_signal": "main",                      // main / maintain / declining
      "diagnosis": [ { "lens": "L4", "severity": "watch", "finding": "...",
                       "evidence_ref": "n_ax3_promo_freq", "implication": "..." } ],

## 9. 常见死法（预填，落 ref 时细化）

- **把不同购买决策塞进同一张图**：移动热点和家用路由器对手不同，混在一起看必错。L0 先按购买决策分桶。
- **嘴硬无据**：严重度未受证据封顶 → 诊断变情绪输出。
- **裁决当结论**：「加大投入」是假设，要 Step 5 实验验证才算 Sourced。
- **生命周期信号当销量**：评论停增 ≠ 真退市，是 Hypothesis，要用户内部数据确认。
- **4P 建议无指向**：每句话必须挂回某条诊断，否则删。
